# Bottleneck-Analyse & Continuous Improvement

## Überblick

Dieser Ordner enthält die Analyse eines identifizierten Engpasses (Bottleneck) im Scrum-Projekt sowie Maßnahmen zur kontinuierlichen Verbesserung.

Ziel ist es, die Auswirkungen von Teamänderungen und reduzierter Aktivität auf den Workflow darzustellen und mögliche Optimierungen aufzuzeigen.

---

## Inhalt

* **Bottleneck-Analyse**
  Beschreibung des Engpasses im Projektverlauf

* **Visualisierungen**

  * Kanban-Board (WIP-Limit erreicht)
  * Metriken zur Aktivität im Repository

* **Continuous Improvement**
  Maßnahmen zur Verbesserung des Workflows

---

## Ziel

* Engpässe im Workflow erkennen
* Prozessprobleme verständlich darstellen
* Verbesserungsmöglichkeiten ableiten
